# Mentor practitioners — the source roster

> **Input for [IDEA-003](ideas/IDEA-003-mentor-layer.md), not the encoded output.** Extracted and
> deduplicated from Ajesh's vision doc (`~/Documents/stuff/founder_mentor_process_map_temple.md`).
> These are **lenses to study and distill**, grouped by the mentor archetype they feed
> (see [MENTORS.md](MENTORS.md)). Per the decided approach — *archetype mentors seeded by named,
> attributed practices* — each name becomes source material for one or more `library/practices/*.md`
> heuristics routed UP via `/boss-learn`. No agent impersonates a person; mentors *cite* the practice.
> The one-line lens below is a study pointer, not the person's endorsement or verbatim view.

## How to use this
1. Pick a practitioner. 2. Distill their **transferable heuristic** (what a founder should *do*, stack-
neutral). 3. `/boss-learn` it UP → `library/practices/<name>.md` (attributed) + a `memory-seed/` feedback
entry if it's a default every project should start with. 4. The mapped mentor agent cites it. Start with
`mentor-venture`'s feeders (it's the cornerstone, already in Quickstart).

### AI-specific UX heuristics (cross-cuts designer + `mentor-humane`)

> Operationalized at runtime, not just in design specs:
> - AI as **options**, not truth (visible confidence; multiple plausible answers)
> - **Human-in-the-loop** for consequential actions (no auto-apply on irreversible)
> - **Visible confidence** — the structured hook output ships this (v0.16+)
> - **Undo / edit / regenerate** — every AI surface needs all three
> - **Deliberate failure states** — what does the user get when the AI is unavailable or wrong?
> - **Brand voice anchoring** — canvas Promises cell IS the design brief; AI defaults are
>   internet-averages (the brand-default problem from [IDEA-010](ideas/IDEA-010-scalable-ai-design.md))
