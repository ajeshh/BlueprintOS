# Humane Product Canvas — BlueprintOS

> A snapshot, not a blueprint. BOSS running its own `/canvas` on itself. Revisit as insight grows.
> Framework: Humane Product Canvas by Ajesh Shah.

## 1 · Human Foundation
| Cell | Answer |
|---|---|
| **People** | Solo founders, indie hackers, product-minded builders using Claude Code — *including first-timers who don't know how to start a company.* What matters to them: momentum, not drowning in setup; learning the craft; keeping options open. |
| **Problem** | Starting is overwhelming. Choices are bad: enterprise scaffolds bury you in ceremony, or a blank repo gives no guidance. No mentor in the room. You reinvent the same scaffolding every project, and hard-won practices stay trapped in one repo. Emotional tension: "I don't really know how to start a company / do this right." |
| **Promises** | Never buried in ceremony you haven't earned; never missing the support you now need. A knowledgeable team-in-a-box shows up *just in time* and grows with you. Momentum + confidence. |

## 2 · Product Expression
| Cell | Answer |
|---|---|
| **Story** | `boss new`, open Claude, `/boss` — a calm, staged guide appears. Capture an idea, keep adding, pressure-test it humanely. As the idea earns it, modes unlock (Quickstart→MVP→V1→Scale) and the right mentors arrive. Specs, design system, board show up at the right time — not before. |
| **Modes of Engagement** | Deterministic CLI (scaffold/registry/sync) + Claude skills & builder agents (intelligence) + mentor agents (counsel). Humane: respects time/attention/autonomy; nothing is forced; *you choose* to level up. |
| **Business Model** | _(open — not yet decided; optionality preserved per Principle 5.)_ Candidates: OSS core + paid mentor/dossier packs; hosted founder dossier; template/practice marketplace. Hard constraint: must align with the promise — never monetize lock-in or ceremony. |

## 3 · Stewardship
| Cell | Answer |
|---|---|
| **Metrics** | Meaningful = projects that actually ship + founders who gain real momentum/confidence (not vanity counts). Activation: first idea captured + canvas filled. Retention: projects that unlock the next mode. Loop health: practices promoted UP into BOSS. |
| **Risks & Harms** | (1) **BOSS bloats into a heavy framework** — betraying its own JIT principle (the most ironic, most likely failure). (2) Imposing ceremony prematurely. (3) False confidence / bad startup or legal advice — mentors must caveat and point to real experts, never replace them. (4) Excluding non-technical founders if too dev-centric. |
| **Principles** | [`PRINCIPLES.md`](../../PRINCIPLES.md) — the six. Humane before viable. |

## 🎯 Incubation heartbeat
- **Riskiest assumption:** that JIT staging + mentors actually *reduce* overwhelm, rather than becoming yet another system to learn.
- **Experiment this week:** dogfood BOSS on BOSS (this), and run one real idea (`margin`) through Quickstart → `/canvas`. Does it feel like help or overhead?
- **What would change the plan:** if modes/mentors feel heavier than a blank repo, ruthlessly simplify — collapse modes or strip ceremony.
