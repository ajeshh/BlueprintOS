# RVW-004 — drive agents from your phone via /remote-control ("be a CEO all day")

## The claim
- **Source:** Reddit r/ClaudeCode (OpinionsRdumb). `docs/research/inbox/remote-control-phone-driven-agents.md`
- **Core assertion:** *Use `/remote-control` to manage 2-3 agents from your phone anywhere (gym,
  meetings, walks); you become "a CEO all day while agents do the grunt work," and checking agent
  updates replaces doomscrolling.*

## Rubric
| # | Question | Finding |
|---|---|---|
| 1 | Contradicts a PRINCIPLE? | **In tension with #6 (humane).** Celebrates swapping the doomscroll compulsion for an agent-update dopamine hit + being "plugged in" at the gym. Not a clean veto, but the humane lens is uneasy. |
| 2 | Evidence grade | **Lowest in the batch.** n=1, breathless ("life-changing," "the future"), wellbeing claims made days in (pure novelty effect), zero data. The feature claim itself is secondhand/unverified. |
| 3 | Duplicate or sharpen? | **Neither.** `/remote-control` is a *Claude Code harness feature* BOSS doesn't own. There is nothing here for BOSS to adopt. |
| 4 | Who serves / harms? | The humane risk: "always-on agent management" can erode attention, rest, and dignity (canvas R&H #5 — cumulative pressure on founders). The "freedom" framing and the harm framing are the same fact. |
| 5 | Cost / ceremony | N/A to BOSS adoption. |

## Verdict: REJECT

Out of scope — a host feature, not a BOSS practice; there is nothing to adopt. Rejected primarily on
that, secondarily on the lowest evidence grade in the batch.

## If REJECT / NOT-YET
- **Why not:** scope (host feature, not BOSS) + evidence (n=1 novelty hype).

## The keeper (recorded reasoning, not a build)

This is exactly the seductive workflow BOSS's conscience exists to be skeptical of, so the verdict is
worth keeping as a **humane stance on the record:** *if BOSS ever coaches founders on working patterns
(mentor-humane / a working-cadence surface), it should NOT endorse "always-plugged-in" agent
management as an unalloyed good.* "Reclaimed attention" and "a new attention trap" are one design away
from each other; the doomscroll→agent-dopamine swap is a lateral move, not a cure. Logged so a future
session doesn't enthuse about phone-driven always-on agents without the humane caveat. (Mirrors how
[[RVW-001]] kept the IDEA-014 finding from a REJECT.)
